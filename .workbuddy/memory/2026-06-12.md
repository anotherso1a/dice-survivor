## 修复项目无法打开

根因：5 个 bug 导致 Godot 编辑器无法加载项目：
1. **`project.godot` `warnings/enable=all`** → 无效值，应为 `true`（Godot 解析项目配置时崩溃）
2. **`constants.gd` 缺少 `class_name Constants`** → run_state.gd / player.gd / enemy_base.gd 中 `Constants.XXX` 全部编译失败
3. **`save_manager.gd` `func load()`** → 与 Godot 内置全局函数 `load()` 冲突，改名为 `load_save()`
4. **`enemy_base.gd` `_died.emit()`** → 信号名是 `died` 不是 `_died`，引用了未定义变量
5. **`player.gd` 第211行缩进错误** → else 分支内的 _label.text 缩进不一致

## 碰撞系统修复（敌人不掉血 + 骑头上）

### Bug 1：ContactDamage.hitbox 引用路径错误
- `NodePath("Hitbox")` 找的是 ContactDamage 的子节点，但 Hitbox 是兄弟节点
- 修复：`NodePath("Hitbox")` → `NodePath("../Hitbox")`

### Bug 2：敌人骑在玩家头上
- 根因：Player 和 Enemy 都是 CharacterBody2D 在同一碰撞层，move_and_slide() 碰撞后把敌人往上滑
- 类幸存者游戏的正确做法：敌人穿过玩家，由 Hitbox Area2D 做伤害检测
- 修复：用碰撞层分离
  - Player: collision_layer=1, collision_mask=0
  - Enemy: collision_layer=2, collision_mask=0
  - Enemy Hitbox (Area2D): collision_mask=1（检测 layer 1 的 Player）

### Bug 3：ContactDamage 每帧扣血
- 原来 `_physics_process` 每帧调用 `body.take_damage()`，60fps = 瞬间秒杀
- 新增 `damage_cooldown: float = 0.5`（@export 可调），命中后进入冷却
- 冷却期间不检测 overlap，break 后只对第一个命中的玩家生效
